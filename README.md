# CUA-Lark 飞书桌面自动化系统 v3.5

让 AI 替你操作飞书！通过自然语言描述任务，系统会自动截图、分析并执行桌面操作。

---

## 🌟 功能特性

### 核心能力
- 🤖 **AI 视觉驱动** - 基于豆包 2.0 Pro（火山方舟），AI 通过截图"看见"屏幕，自主决策下一步操作
- 🖱️ **完整桌面操作** - 鼠标点击/拖拽/滚轮、键盘输入/热键、文件操作、窗口控制，覆盖所有桌面交互
- 🔄 **闭环执行** - 截图 → AI 分析 → 执行操作 → 再截图检查，循环直到任务完成
- 💬 **连续对话** - 任务间保留对话历史，AI 记得之前做过什么，支持多轮交互
- ⏸️ **暂停与继续** - 随时暂停任务手动介入，继续按钮一键恢复，不丢失上下文
- 🙋 **智能求助** - AI 遇到困难自动暂停并请求帮助，用户补充信息后继续执行

### 智能交互
- 🧠 **任务理解确认** - 每次任务开始前 AI 先分析需求、识别缺失信息、展示理解，用户确认后再执行
- 📝 **文本资料处理** - AI 可复制屏幕上的文本资料到 CUA 界面，请求用户发送给 AI 分析
- 🎯 **快捷连招** - `click_paste` 一步完成点击+粘贴，`select_text` 一步完成点击+全选
- 🖥️ **窗口控制** - AI 可通过 `show_cua_window` / `hide_cua_window` 指令控制 CUA 主窗口显示/隐藏

### 界面与体验
- 🎛️ **迷你控制条** - 任务执行时主窗口隐藏，左下角显示迷你控制条实时监控状态，不干扰 AI 视觉判断
- 📊 **实时状态显示** - 当前步骤、当前操作、执行日志实时更新，一目了然
- � **图形化配置** - 内置 API 配置窗口，预设火山引擎等常用地址，首次使用自动引导
- 📖 **内置说明书** - 右侧面板显示完整操作指南，随时查阅

### 安全与稳定
- 🛡️ **多重安全网** - 连续相同操作自动检测（超 3 次停止）、指数退避重试（429 限流）、最大迭代限制（250 次）
- � **故障安全** - 鼠标移到屏幕左上角紧急停止（pyautogui 内置）
- 📡 **API 稳定** - 自动重试机制，网络波动不影响任务执行
- � **全程可控** - 暂停、继续、终止，用户始终掌握主动权

### 核心工作流程
```
用户输入任务 → AI理解确认 → 循环执行: 截图 → AI决策 → 执行操作 → 检查结果
                                              ↑                                      │
                                              └──────── 用户可随时暂停/补充信息 ←─────┘
```

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

依赖项：`pyautogui` `pyperclip` `requests`

### 2. 配置 API

运行程序后首次使用会自动弹出配置窗口，或点击界面的 **🔧 API配置** 按钮：

1. 选择 API 预设（推荐"火山引擎（豆包）"，自动锁定地址）
2. 填入 **API Key** 和 **接入点 ID**
3. 点击 **💾 保存**

> ⚠️ 配置保存在 `config_api.json`，不会提交到代码仓库。

### 3. 运行程序

| 方式 | 命令 | 说明 |
|---|---|---|
| 图形界面（推荐） | `python gui_app.py` | 带实时日志和控制面板 |
| 终端模式 | `python feishu_cua.py "你的任务"` | 命令行直接执行 |

### 4. 使用示例

```
帮我预约一个明天下午3点的会议，主题是"周报讨论"
```

```
在云文档里新建一个表格文档
```

```
你好，给我讲个笑话          ← 纯聊天，不需要操作电脑
```

```
│  │   ├─ 隐藏窗口 → 截图                 │
│  │   ├─ send_to_ai() → 获取AI指令       │
│  │   ├─ 执行操作                        │
│  │   └─ 保存对话历史                     │
│  └─ 连续操作检测 / 终止信号检查           │
├─────────────────────────────────────────┤
│  ai_communicator.py  AI通信层           │
│  ├─ send_to_ai(): 发送截图+任务给API     │
│  ├─ understand_task(): 任务理解确认      │
│  └─ call_api_with_retry(): 指数退避重试  │
├─────────────────────────────────────────┤
│  command_parser.py  命令执行层           │
│  └─ parse_and_execute_command()          │
│      └─ 将JSON指令翻译成具体桌面操作      │
└─────────────────────────────────────────┘
```

## 📋 AI 支持的操作

### 鼠标操作
```json
{"action":"left_click","x":0.500,"y":0.300}     // 左键点击（坐标为屏幕比例0~1）
{"action":"right_click","x":0.500,"y":0.300}     // 右键点击
{"action":"double_click","x":0.500,"y":0.300}    // 双击
{"action":"drag","start_x":0.1,"start_y":0.1,"end_x":0.5,"end_y":0.5}  // 拖拽
{"action":"scroll","clicks":5,"x":0.800,"y":0.500}  // 滚轮
```

### 键盘 & 输入
```json
{"action":"click_paste","x":0.500,"y":0.300,"text":"内容"}  // 输入文本（唯一指定方式！点击+粘贴）
{"action":"select_text","x":0.500,"y":0.300}        // 选中文本（点击+ctrl+a连招，用于替换已有内容）
{"action":"type_text","text":"替换内容"}             // 替换文本（必须与select_text配合使用）
{"action":"press_key","key":"enter"}               // 单按键
{"action":"hotkey","keys":["ctrl","a"]}            // 组合热键
```

### 文件 & 窗口
```json
{"action":"open_folder","path":"C:\\Users\\Desktop"}  // 打开文件夹
{"action":"show_cua_window"}                         // 显示CUA主窗口（无需参数）
{"action":"hide_cua_window"}                         // 隐藏CUA主窗口（无需参数）
{"action":"hotkey","keys":["win","d"]}                // 返回桌面
```

### 任务控制
```json
{"action":"task_completed","result":"已完成！"}       // 任务完成
{"action":"request_input","prompt":"请补充信息"}       // 请求用户帮助
```

## 💡 使用技巧

### 写好任务描述
- 说清楚**目标**："预约明天下午3点的会议"
- 给出**具体信息**：时间、人员、文件名
- 如果有具体路径，用 `open_folder` 比 AI 视觉找更准

### AI 卡住了怎么办
1. 系统会自动检测连续重复操作，强制请求你帮助
2. 你也可以随时点**暂停按钮**暂停，然后在输入框补充信息后继续
3. 补充的信息会写入对话历史，AI 会记得

### 文本资料处理
如果任务涉及处理文档内容：
1. AI 会尝试选中屏幕上的资料并复制
2. AI 调用 `show_cua_window` 调出CUA主窗口
3. AI 将资料粘贴到输入栏，发送 `request_input`
4. 你点击继续后，AI 就能分析资料内容并据此执行任务

## 🛡️ 安全机制

| 机制 | 说明 |
|---|---|
| **故障安全** | 鼠标移到屏幕左上角紧急停止（pyautogui内置） |
| **最大迭代次数** | 默认 250 次，防止无限循环 |
| **连续操作检测** | 相同操作超过 3 次自动停止并请求帮助 |
| **指数退避重试** | API 429 限流时自动等待重试（最多 5 次） |
| **暂停按钮** | 随时可停，不丢失上下文 |

## ❓ 常见问题

**Q: 程序怎么停止？**
A: 点 ⏸ 暂停按钮，或把鼠标移到屏幕左上角。暂停后可手动介入操作，点 ▶ 继续恢复执行。

**Q: AI 一直点错位置怎么办？**
A: 点 ⏸ 暂停，手动帮它点了正确位置，然后直接点 ▶ 继续（或输入补充信息后继续）。

**Q: 可以连续给多个任务吗？**
A: 可以，对话历史会保留，AI 记得之前的上下文。

**Q: 支持哪些 AI 模型？**
A: 目前适配火山引擎（豆包），通过 `config_manager.py` 可扩展其他兼容 OpenAI 接口的模型。

**Q: CUA 窗口挡住视线怎么办？**
A: 任务执行时主窗口会自动隐藏，只显示左下角的迷你控制条，不影响 AI 判断。

**Q: AI 怎么调出 CUA 主窗口？**
A: AI 可以通过 `show_cua_window` 指令调出主窗口，比如在需要粘贴文本资料到输入栏时。
